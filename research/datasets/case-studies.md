## 1. Classic 3-Act Framework

Used by: Most agencies, software houses, general use

### Structure
```
ACT 1: The Challenge (20%)
  • Client background
  • The problem they faced
  • Previous attempts and failures
  • Why they came to us

ACT 2: The Solution (50%)
  • Our approach/methodology
  • The solution we built
  • Key decisions and tradeoffs
  • Implementation process

ACT 3: The Results (30%)
  • Quantitative outcomes
  • Qualitative outcomes
  • Client testimonial
  • Long-term impact
```

### English Example
```
[Client Name] — [Industry] — [Project Type]

THE CHALLENGE
[Client], a leading [company type] in [country], was struggling with [problem]. 
Their existing [system/process] was causing [pain point 1], [pain point 2], 
and [pain point 3]. Previous attempts to solve this [failed/didn't scale].

THE SOLUTION
We deployed our [methodology/platform] across [scope]. 
Phase 1 focused on [key deliverable]. 
Phase 2 added [additional capability].
The team collaborated via [communication method] with [frequency].

THE RESULTS
• [Metric 1]: [X]% improvement
• [Metric 2]: [X]% cost reduction
• [Metric 3]: [X] hours saved per week
• [Metric 4]: [X] user satisfaction score

"[Client quote highlighting impact]"
— [Name], [Title] at [Client]
```

### Arabic Example
```
[اسم العميل] — [القطاع] — [نوع المشروع]

التحدي
[العميل]، شركة [نوع] رائدة في [البلد]، كان يعاني من [المشكلة].
نظامهم/عمليتهم الحالية كانت تسبب [نقطة الألم 1]، [نقطة الألم 2]، و [نقطة الألم 3].
المحاولات السابقة لحل هذه المشكلة [فشلت/لم تتوسع].

الحل
طبقنا [منهجيتنا/منصتنا] عبر [النطاق].
المرحلة 1 ركزت على [تسليم رئيسي].
المرحلة 2 أضافت [قدرة إضافية].
الفريق تعاون عبر [طريقة التواصل] بـ [التكرار].

النتائج
• [المؤشر 1]: تحسن [نسبة]%
• [المؤشر 2]: تخفيض [نسبة]% في التكاليف
• [المؤشر 3]: توفير [عدد] ساعة في الأسبوع
• [المؤشر 4]: [نسبة]% رضا المستخدمين

"[اقتباس العميل يسلط الضوء على الأثر]"
— [الاسم]، [المنصب] في [العميل]
```

## 2. Challenge → Solution → Result Framework

Used by: SaaS companies, enterprise software

### Structure
```
Header:
  Client name + industry + location + time to completion

Challenge Section:
  • Context (who, where, what business)
  • Pain point description (what was broken)
  • Business impact of the problem ($X lost, X% inefficiency)

Solution Section:
  • Our product/service implemented
  • How it addressed each pain point
  • Implementation timeline (usually 4-12 weeks)
  • Key features utilized

Result Section:
  • Measurable outcomes (table format preferred)
  • Before/after comparison
  • Quote from client decision-maker
  • Future outlook
```

### Metrics Table Format
```
| Metric            | Before           | After          | Improvement |
|-------------------|------------------|----------------|-------------|
| Process time      | 8 hours          | 45 minutes     | 89% faster  |
| Error rate        | 12%              | 0.5%           | 96% reduction|
| Customer tickets  | 200/month        | 25/month       | 87% fewer   |
| Team productivity | 60%              | 95%            | 35% increase|
```

## 3. Before → After → Bridge Framework

Used by: Branding agencies, UI/UX firms, design-led companies

### Structure
```
Screenshot/Visual Hook
  • Before image (left) vs. After image (right)

The "Before" Section
  • What the client had before
  • Why it wasn't working
  • User/customer complaints

The Bridge (Our Process)
  • How we got from before to after
  • Key design/strategy decisions
  • Client collaboration points

The "After" Section
  • The new solution in detail
  • User reception/feedback
  • Business results
```

### Visual Emphasis
| Element | Format | Detail |
|---|---|---|
| Hero image | Side-by-side mockup | Full width, annotated |
| Process visualization | Flow diagram | 3-5 steps |
| Result visualization | Dashboard screenshot | Annotated with metrics |
| Client quote | Pull quote | Styled differently |

## 4. The Narrative Journey Framework

Used by: Marketing agencies, branding firms, storytelling-focused companies

### Structure
```
Chapter 1: The Setup
  • Client's origin story
  • Market position before engagement
  • The moment they realized change was needed

Chapter 2: The Turning Point
  • Discovery workshop outcomes
  • The "aha" moment
  • Strategic pivot or decision

Chapter 3: The Transformation
  • The work itself
  • Milestones along the way
  • Team collaboration highlights

Chapter 4: The New Reality
  • Life after the project
  • Measurable outcomes
  • Client reaction and emotion

Epilogue: What's Next
  • Ongoing relationship
  • Future plans
```

## 5. Metrics-First Framework

Used by: SEO agencies, performance marketing, data-driven companies

### Structure
```
THE NUMBER
[Biggest metric] — e.g., "300% increase in organic traffic"

CLIENT AT A GLANCE
• Company: [Name]
• Industry: [Sector]
• Location: [Country]
• Timeframe: [Duration]
• Services: [List]

THE BASELINE
• Starting metrics before engagement
• Industry benchmarks
• Competitor context

THE STRATEGY
• Approach taken
• Channels/tactics used
• Timeline of execution

THE BREAKTHROUGH
• Month-by-month or milestone-by-milestone results
• Key inflection points
• Organic growth trajectory

THE FINAL NUMBERS
• [Metric 1]: [X] → [Y] ([Z]% change)
• [Metric 2]: [X] → [Y] ([Z]% change)
• [Metric 3]: [X] → [Y] ([Z]% change)
• ROI: [X]x return on investment

THE VERDICT
[Client testimonial focused on numbers]
```

## 6. Video Case Study Structure

Used by: All (especially agencies wanting high engagement)

### Structure
```
0:00-0:15 — Hook
  • Problem statement or dramatic result

0:15-0:30 — Client Introduction
  • Who they are, what they do

0:30-1:15 — The Challenge
  • Client describes the problem in their own words
  • Visuals of "before" state

1:15-2:30 — The Solution
  • How we approached it
  • Key features/moments
  • B-roll of team/product

2:30-3:30 — The Results
  • Client describes outcomes
  • Metric overlay graphics
  • Before/after comparison

3:30-4:00 — Future & CTA
  • Client on future plans
  • "Ready for your own success story?" + CTA
```

### Video Length Analysis
| Length | Engagement | Best For |
|---|---|---|
| 60-90 seconds | Highest | Social media, homepage |
| 2-3 minutes | High | Website case study page |
| 3-5 minutes | Medium | Sales enablement |
| 5-10 minutes | Low-Medium | Technical deep dive |

## 7. Technical Deep-Dive Framework

Used by: AI companies, developer tools, cloud providers

### Structure
```
Executive Summary (1 paragraph)
  • What was built and why

Architecture Overview
  • System diagram
  • Technology stack
  • Integration points

The Problem (Technical)
  • Scale challenges
  • Performance bottlenecks
  • Migration complexity

Our Technical Approach
  • Key architectural decisions
  • Why we chose specific technologies
  • Tradeoffs and considerations
  • How we handled [edge cases]

Implementation Details
  • Code snippets (if relevant)
  • Configuration examples
  • Performance tuning

Results (Technical)
  • Latency improvements
  • Throughput increases
  • Cost optimization
  • Reliability metrics

Key Learnings
  • What worked well
  • What we'd do differently
  - Recommendations for similar projects
```

## 8. Partnership Story Framework

Used by: Agencies emphasizing long-term relationships

### Structure
```
HOW WE MET
  • First contact story
  • Initial conversation highlights
  • Why they chose us

THE GROWTH PHASE
  • First project scope
  • Results that led to more trust
  • Expansion of relationship

THE SCALE PHASE
  • Current scope of work
  • Number of projects completed
  • Team structure (dedicated team, etc.)

THE RESULTS (AGGREGATE)
  • Cumulative metrics across all projects
  • Value delivered over the relationship
  • Awards or recognition

THE RELATIONSHIP TODAY
  • Current partnership status
  • Ongoing initiatives
  • Client quote on partnership
```

### English Example — Partnership
```
[Client] and [Company] — [X] Years of Partnership

HOW WE MET
In [Year], [Client] needed [specific need]. They had tried [number] 
other agencies, but none understood the Saudi market. Our first 
meeting lasted [time] — and we knew it was the start of something.

THE GROWTH PHASE
Project 1: [Scope] → [Result]
Project 2: [Scope] → [Result]
Project 3: [Scope] → [Result]

THE SCALE PHASE
Today, we serve as [Client]'s primary [service] partner,
with [number] dedicated team members. We've delivered 
[number] projects totaling [metric] of impact.

THE RELATIONSHIP
"You're not our vendor — you're our technology department."
— [Client Name], [Title]
```

## 9. The ROI Calculator Framework

Used by: SaaS, enterprise sales, cloud providers

### Structure
```
THE INVESTMENT
  • Project cost (can be shown as range)
  • Implementation time
  • Team effort required

THE RETURN (Itemized)
  • Cost savings: $X/year
  • Time savings: X hours/year
  • Revenue increase: $X/year
  • Error reduction: X% fewer mistakes
  • Employee productivity: X% increase

THE BREAK-EVEN
  • Time to break even: X months
  • Month 1-3 ROI: X%
  • Month 4-6 ROI: X%
  • Year 1 ROI: X%
  • Year 3 ROI: X%

TOTAL 3-YEAR IMPACT
  • Total value delivered: $X
  • ROI multiple: Xx
  • Net benefit: $X

THE VERDICT
[Short client statement on ROI]
```

## 11. Arabic Case Study Localization Patterns

### Language Positioning
| Element | Arabic-First | Translated | Bilingual |
|---|---|---|---|
| Headline | Arabic headline unique | Translated from EN | Arabic primary, EN secondary |
| Body | Written originally in AR | Machine/post-edited | Separate AR/EN versions |
| Quotes | Original Arabic quotes | Translated | Original Arabic + translation |
| Metrics | AR numerals (شرق أوسطية) | Western numerals | Consistent format |
| CTA | Arabic CTA (first) | Same CTA | AR primary/EN secondary |

### Cultural Elements in Arabic Case Studies
| Element | Frequency | Example |
|---|---|---|
| Religious references | 30% | "بفضل الله ثم..." (By the grace of God...) |
| National pride | 60% | "تماشياً مع رؤية المملكة" (In line with the Kingdom's vision) |
| Relationship emphasis | 45% | "علاقة شراكة حقيقية" (A true partnership) |
| Trust language | 55% | "نضع ثقتنا في..." (We place our trust in...) |
| Hospitality language | 25% | "استقبلونا بأفضل ترحيب" (They welcomed us warmly) |

### Format Preferences
| Format | Arabic Reader Preference | Reason |
|---|---|---|
| Long-form narrative | Preferred | Storytelling tradition |
| Bullet points | Moderate | Less common but accepted |
| Tables with numbers | High | Expectation of clarity |
| Visual results (charts) | Very high | Prefer visual over text numbers |
| Client photo (identifiable) | Conditional | Privacy-conscious culture |
| Government logo display | Highly trusted | Authority credibility |
