## 9. FAQ Page Structure Patterns

### Pattern 1: Accordion Expansion
```
Structure: Category tabs → Click question → Expand answer
Popular on: SaaS websites, developer documentation
Example sites: Salla, Foodics, Unifonic
Pros: Clean, scannable, good for 10+ questions
```

### Pattern 2: Search + Categories
```
Structure: Search bar → Category filter → Question list
Popular on: Enterprise sites, knowledge bases
Example sites: Elm, stc Cloud
Pros: User-controlled discovery, good for 30+ questions
```

### Pattern 3: Simple List (Collapsed)
```
Structure: Questions in list → Click to reveal
Popular on: Small agencies, startups
Example sites: Smaller software houses
Pros: Simple, quick implementation, mobile-friendly
```

### Pattern 4: Detailed Answer Pages
```
Structure: Category page → Each question is a separate page
Popular on: Large enterprises, support portals
Example sites: Big SaaS platforms
Pros: SEO-optimized, detailed answers, trackable
```

### FAQ Page Design Elements
| Element | Prevalence | Example |
|---|---|---|
| Accordion expand/collapse | 75% | "Click to show answer" |
| Category tabs | 50% | General · Technical · Pricing |
| Search bar | 35% | "Search FAQs..." |
| Contact CTA after FAQs | 60% | "Still have questions? Contact us" |
| Live chat alongside FAQs | 40% | Bottom-right chat widget |
| "Was this helpful?" feedback | 25% | Thumbs up/down on each answer |
| Related articles | 20% | "You might also want to know..." |

## 10. FAQ Writing Style Analysis

### Answer Tone Distribution
| Tone | Prevalence | When Used |
|---|---|---|
| Direct & concise | 50% | Simple factual questions |
| Empathetic + solution | 25% | Pricing, cancellation, complaints |
| Technical + detailed | 15% | Integration, API, security |
| Legal/compliance language | 10% | Data privacy, regulations |

### Answer Structure Patterns
| Structure | Example | Frequency |
|---|---|---|
| Yes/No + Detail | "Yes. We are ZATCA compliant. Here is our certificate..." | 40% |
| Step-by-step | "The process has 3 phases: Discovery, Build, Launch." | 25% |
| Comparison | "Unlike traditional agencies, we use agile methodology..." | 15% |
| Myth-busting | "No, you don't need a large team. Here's why..." | 10% |
| Storytelling | "One client asked us the same thing. Here's what happened..." | 10% |
